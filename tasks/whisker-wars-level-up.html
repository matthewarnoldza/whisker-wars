🐾 Whisker Wars → Steam-Worthy

Full multi-agent review & improvement plan · 6 specialist reviews · 55 findings, each high-impact claim adversarially verified against the code · 3 July 2026

TL;DR

The game's bones are better than expected — the packaging is what's holding it back. React fundamentals are solid (correct Zustand selectors, memoized Collection, React.memo'd cards), the core loop is readable, and 16 achievements + full offline play mean Steam integration is closer than it looks. But five things keep it at "entry-level browser game": (1) 193MB of raw, unresized PNG art ships byte-for-byte into every build; (2) there is no desktop identity — mobile-first centered column at 1080p, emoji as the entire icon system, no settings screen, thin combat feedback; (3) zero desktop platform layer (no wrapper, fullscreen, controller, Steamworks); (4) the economy and roguelike are tuned for casual web sessions, not a pay-once 10+ hour audience (runaway coin inflation, 12 boons, 2 bosses, rubber-band scaling); (5) the riskiest code — save/load and combat — has no tests, no save versioning, and lives inside 1,900-line components. There is also a live revenue leak: the paid expansion unlocks from a URL parameter with no verification. The plan below sequences ~10–13 weeks of work into six phases, with four decisions needed from you first.

⛔ Fix now — independent of everything else (~2–3 days)

  • Revenue leak: appending ?payment=success&view=jungle to the URL permanently unlocks the paid R65 expansion — App.tsx:379 sets junglePassUnlocked: true with zero verification. The fix is nearly free: checkPaymentStatus() at store.ts:1585 already reads the webhook-written payment record — wire it into this path before trusting the param. verified
  • Save codes are guessable & unauthenticated: 8 prefixes × 9,000 numbers = ~72k total codes (cloudSave.ts:55), and downloadSave fetches any save by code with no owner check — anyone can enumerate and read or overwrite other players' saves. Widen the keyspace and add a per-save owner token. verified
  • Accessibility bug: prefers-reduced-motion silently mutes all sound effects — a stray guard at sound.ts:33 returns early from playSound(). Motion and audio are orthogonal; delete the guard. verified
  • Meta Pixel ships in every build: the Vite strip plugin removes only GA and only in itch.io mode — the Facebook Pixel (index.html:15) ships to web, itch.io, and mobile, and the Privacy Policy doesn't disclose it. Strip it from non-web builds and update the policy. verified

1 · Desktop wrapper for Steam

✓
Recommended
Tauri v2
~3–6MB installer using the OS webview. The itch.io build mode already emits the relative-path, tracker-stripped bundle a webview shell consumes. Steamworks via steamworks-rs Tauri command. Best Steam Deck behavior.
✓
Electron
~150MB+ but guarantees identical Chromium rendering on every machine and the mature greenworks Steamworks binding. Choose if webview inconsistency (older Windows WebView2) worries you.

2 · Jungle of Talons on Steam

✓
Recommended
Bundle into base price
Force junglePassUnlocked = true behind a VITE_BUILD_TARGET==='steam' flag (4 sites in store.ts), delete the Yoco path from the bundle. Simplest; a content-complete game justifies a higher base price.
✓
Paid Steam DLC
Replace the boolean with BIsDlcInstalled(dlcAppId) via the Steamworks bridge. A second SKU and store page to maintain, but preserves the two-tier pricing you have on web.

3 · Battle engine refactor timing

✓
Recommended
Extract engine before content push
Pull combat out of the 1,921-line BattleArena into a pure, seeded, testable reducer in src/game/battle first. Every content item after (boons, bosses, abilities) lands once instead of twice, and combat juice hooks into engine events cleanly.
✓
Polish first, refactor later
Ship visible improvements sooner; accept that the 30–40 boon expansion and new bosses get built twice (BattleArena + JungleBattle) and drift. Only sensible if you want a fast marketing beat.

4 · Icon & visual identity investment

✓
Recommended
Commission a cohesive icon set
Coins, rarities, elements, status effects, nav — one artist pass (~30–40 icons) so the game looks identical on every OS. This is the single biggest "browser game" tell (307 emoji as functional UI).
✓
Restyle an open SVG library
Tint/re-stroke a permissively-licensed set (e.g. game-icons.net, CC-BY) to the gold/purple palette. Cheaper and immediate, less distinctive; credit required.

The five pillars

1
Shrink the game by 85%. 145 PNGs at 864×1184 render in 208×320 boxes. A sharp-based prebuild (resize to 2× display + WebP q80, <picture> fallback) takes 193MB → ~25–35MB. The script already exists at scripts/optimize-images.mjs — it's dead code because sharp was never installed. This one change transforms load time on web, itch.io zip size, and the Steam depot.
2
Harden the foundation. A versioned save schema with one source-of-truth serializer (today the ~27-field list is hand-copied in 4 places and has already caused documented data loss), a pure seeded battle engine shared by both arenas, store split into domain slices, and tests on exactly the code that has bitten you before.
3
Build a desktop identity. A designed 16:9 stage instead of a centered mobile column, real icons instead of emoji, a Settings screen (volume sliders exist in code but have no UI), self-hosted fonts, a modal queue instead of four timed popups on load, and combat juice: hitstop, stage shake, impact flash, damage numbers anchored to the card that was hit.
4
Ship the platform layer. Tauri/Electron wrapper consuming the existing itch.io build mode, a steam build target (bundle expansion, strip trackers, kill doomed network loops), Steamworks achievements (16 already computed in the store), Steam Cloud saves, fullscreen/quit/pause, keyboard + controller navigation for Deck.
5
Tune for a pay-once audience. Real coin sinks before income runs away (a late dog pays ~2,100/fight; full ascension costs 3,500), 30–40 synergistic boons and 4–5 bosses instead of 12/2, replace rubber-band scaling with an opt-in ascension ladder, dupe pity for the 9-copy Prismatic grind, and unshackle training/events from the real-world clock.
